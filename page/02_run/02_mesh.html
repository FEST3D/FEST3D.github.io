To solve a real-life problem, like flow over an airfoil, first, a domain of computation has to be defined. This domain represents the system of interest which we like to study. Now to apply a numerical method to solve the governing equation (Navier-Stokes) on this domain, it needed to be discretized. This process is not part of the FEST-3D code and it assume a predefine discretized domain(grid) will be provided as input. The FEST-3D expect structured grid in the following format

Imax  Jmax  Kmax
X(1,1,1)   Y(1,1,1)   Z(1,1,1)
X(2,1,1)   Y(2,1,1)   Z(2,1,1)
X(3,1,1)   Y(3,1,1)   Z(3,1,1)
...
X(Imax,1,1)   Y(Imax,1,1)   Z(Imax,1,1)
X(1,2,1)   Y(1,2,1)   Z(1,2,1)
X(2,2,1)   Y(2,2,1)   Z(2,2,1)
...
...
X(Imax,Jmax,1)   Y(Imax,Jmax,1)   Z(Imax,Jmax,1)
X(1,1,2)   Y(1,1,2)   Z(1,1,2)
X(2,1,2)   Y(2,1,2)   Z(2,1,2)
X(3,1,2)   Y(3,1,2)   Z(3,1,2)
...
...
...
X(Imax,Jmax,kmax)   Y(Imax,Jmax,kmax)   Z(Imax,Jmax,kmax)

In the case of multi-block, as shown in Fig. 1, each process requires a separate grid file and block should be created such that every face of that block employs only a single boundary condition. For now, two different types of boundary conditions at a single face of a block is not supported by the FEST-3D.
